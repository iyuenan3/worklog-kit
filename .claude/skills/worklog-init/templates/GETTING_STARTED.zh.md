## 今晚（第一次记录）

睡前对 Claude Code 说「记录今天」，**建议附带两三句今天做了什么**，例如：

> 记录今天。上午主要在改 X 项目的登录流程，下午跟人对了一版需求，晚上没干活。

你说的话就是全部输入，说完即可去睡：ingest 全程无人值守，任何数据源失败都只记一句、绝不停下来等你。

## 明早（第一次复盘）

1. 读 `diaries/` 下的第一篇日记
2. 哪里不对，直接说「更新日记，把 XXX 改成 YYY」
3. 看一眼终端里的晨起报告：有没有源被跳过、有没有「新发现项目待定级」

## 常见首跑问题

- **某个项目没被扫到** → 检查 `worklog.config.yaml` 的 `sources.roots` 是否覆盖它的路径
- **时间线是空的** → 那天确实没有 commit；把工作口述给 brain-dump 即可
- **新项目只有一行计数** → 这是安全默认（presence 级），说一句「X 项目升为 detail」即可
- **不想让某项目出现在日记里** → 在该项目根放一个 `.worklogignore` 空文件
- **项目用符号链接组织** → 自动发现不跟随 symlink，把真实路径加进 `sources.roots` 即可

## 三个习惯

1. 睡前一句话 brain-dump（扫描有盲区，你的一句话是最强的数据源）
2. 新想法先扔 `inbox/`，别直接写进 wiki（让 ingest 消化）
3. wiki 结构想改就改，改完说一声让 AIREADME 记下为什么
